🟢 Computer Architecture

Stage 1: Conceptual Exam Keywords (Bilingual Cheat Sheet)

1. DRAM vs. SRAM

Features SRAM DRAM
Cost Expensive Cheaper
Speed Faster Slower
Refresh No need refreshing Need refreshing
Use Case Cache Memory (L1, L2, L3) Main Memory
Cell Structure 6 transistors for every 1-bit 1 transistor + 1 capacitor for every 1-bit
Density (Yoğunluk) Less Dense (Large Cells) More Dense (Small Cells)
Heat Generation High Low
Power Consumption High (During operation) Constant (Due to refresh)

2. Locality of Reference

  • Temporal Locality: If accessed, highly likely to be accessed again soon (e.g., loop variables). Cache keeps recently accessed data.
    🇹🇷 Zamansal Yerellik: Şu an kullanılan bir veriye yakın zamanda tekrar ihtiyaç duyulmasıdır (örn: Döngülerdeki sayaçlar). Bu yüzden Cache, son kullanılan verileri silmez, içinde tutar.
  • Spatial Locality: If accessed, highly likely that nearby locations will be accessed soon. Memory brings data in Blocks/Lines.
    🇹🇷 Konumsal Yerellik: Bir veriye erişildiğinde, adres olarak onun hemen yanındaki verilere de sırayla erişilme ihtimalidir (örn: Diziler/Arrays). Bu yüzden bellekten veri tek tek değil, "Blok" halinde toptan getirilir.

3. Cache Writing Policies

  • Write-Through: Written to cache and main memory simultaneously. Ensures strict consistency but slower performance.
    🇹🇷 Anında Yazma: Veri aynı anda hem Cache'e hem Ana Belleğe yazılır. Veriler her an birbiriyle tutarlıdır ama ana belleği beklemek sistemi yavaşlatır.
  • Write-Back: Written only to cache initially (marked with a dirty bit). Written to main memory only upon eviction. Faster execution.
    🇹🇷 Gecikmeli Yazma: Veri sadece Cache'e yazılır ve "kirli" işaretlenir. Ancak o veri Cache'ten silineceği (eviction) zaman toptan ana belleğe aktarılır. Çok daha hızlıdır.

4. The Three I/O Methods

  • Programmed I/O: CPU repeatedly checks status (polling). Wastes CPU cycles.
    🇹🇷 Programlı I/O: İşlemci sürekli "işin bitti mi?" diye donanımı kontrol eder (Polling). İşlemcinin vaktini çok boşa harcar.
  • Interrupt-Driven I/O: CPU continues executing other tasks. Device sends an interrupt when ready.
    🇹🇷 Kesme Tabanlı I/O: İşlemci komutu verir ve kendi işine döner. Donanım hazır olduğunda işlemciye bir sinyal (Interrupt) göndererek haber verir.
  • DMA (Direct Memory Access): DMA controller handles transfer directly between I/O and Memory, bypassing CPU.
    🇹🇷 Doğrudan Bellek Erişimi: Büyük veri transferlerinde (örn: SSD, Ağ kartı) işlemciyi hiç yormadan, transfer işini doğrudan bellek ile donanım arasında yapan özel kontrolcüdür.

5. Interrupt Priority & Nesting

  • Priority: Assigning importance levels. CPU services the one with the highest priority first.
    🇹🇷 Öncelik: Aynı anda birden fazla kesme (interrupt) gelirse, işlemcinin daha hayati olanı (yüksek öncelikliyi) seçip önce onu işlemesidir.
  • Nesting: A higher-priority interrupt is allowed to preempt execution of a lower-priority routine.
    🇹🇷 İç İçe Geçme: İşlemci düşük öncelikli bir işi yaparken araya acil bir iş girerse, mevcut işi duraklatıp acil olanı halletmesi ve sonra geri dönmesidir.

6. Thrashing & Cache Mapping

  • Thrashing: Performance collapse where the CPU spends more time swapping blocks (due to conflict misses) than executing instructions.
    🇹🇷 Thrashing (Patinaj/Tıkanma): Direct-Mapped cache'lerde farklı verilerin aynı yere gitmeye çalışması yüzünden birbirlerini sürekli silmesi ve işlemcinin sadece veri yüklemekle vakit kaybetmesidir.
  • Set-Associative Cache: Groups multiple lines into a "Set". Allows multiple blocks mapping to the same index to co-exist, reducing conflict misses.
    🇹🇷 Set-Associative (Küme İlişkili) Çözümü: Adresleri tek kişilik odalar yerine çok kişilik "Kümeler" haline getirerek, aynı adrese denk gelen farklı verilerin birbirini silmeden yan yana yaşamasını sağlar. Thrashing'i çözer.

7. Instruction Cycle with Interrupts (Kesmeler)

  • Core Concept: The CPU is extremely fast, but hardware devices are slow. Interrupts were invented so the CPU doesn't waste time waiting — it does other work and gets notified when the device is ready.
    🇹🇷 Temel Mantık: İşlemci çok hızlıdır, donanımlar yavaştır. İşlemci donanımı boş boş beklemesin diye "Kesme" (Interrupt) icat edilmiştir. Anahtar kelimeler: Improve efficiency, immediate attention.
  • Interrupt Types (H.I.T.P.):
    Hardware — Hardware malfunction, power failure
    I/O — Signals from keyboard, mouse, printer
    Timer — Routine tasks from the system clock
    Program — Software errors like division by zero, overflow
    🇹🇷 Şifre H.I.T.P.: Hardware (Donanım arızası), I/O (Klavye/fare sinyalleri), Timer (Saat görevleri), Program (Sıfıra bölme gibi yazılım hataları).
  • Modified Cycle: Fetch → Execute → Interrupt Check. After each instruction, the CPU checks if any interrupt has arrived.
    🇹🇷 Değişen Döngü: Her komut çalıştırıldıktan sonra işlemci bir kesme gelip gelmediğini kontrol eder.
  • Two Interrupts at Once?
    Sequential: Disabling interrupts — new interrupt is queued and handled after the current one finishes.
    Nested: Priority-based — a higher-priority interrupt (e.g., Hardware) can preempt a lower-priority one (e.g., I/O).
    🇹🇷 İki Kesme Aynı Anda: Sequential = Yeni geleni sıraya koyar. Nested = Önceliği yüksek olan düşük olanı böler.

8. Interconnection Structures (PCIe, QPI)

  • Core Concept: Old systems used a Shared Bus (single road for everything → traffic jams). Modern systems use Point-to-Point connections (PCIe, QPI) where each device has its own dedicated highway.
    🇹🇷 Temel Mantık: Eski sistemlerde her şey tek yolu paylaşıyordu ve trafik sıkışıyordu. Yeni sistemlerde her donanımın işlemciye özel otobanı vardır. Neden geçtik? No bottlenecks!
  • Point-to-Point Features (M.P.L.):
    Multiple direct connections
    Packetized data (data is sent in packets)
    Layered protocol (layered architecture)
    🇹🇷 Şifre M.P.L.: Multiple direct connections (Çoklu doğrudan bağlantı), Packetized data (Paketler halinde iletim), Layered protocol (Katmanlı mimari).
  • PCIe Layers (P.D.T.):
    Physical Layer — Physical cables/wires
    Data Link Layer — Packet ordering and error control
    Transaction Layer — Packet creation
    🇹🇷 Şifre P.D.T.: Physical (Fiziksel kablolar), Data Link (Paket sırası ve hata kontrolü), Transaction (Paket yaratma).
  • QPI: Used for high-speed connections between CPU ↔ CPU or CPU ↔ Memory Controller.
    🇹🇷 QPI: CPU ile CPU arası veya CPU ile RAM kontrolcüsü arasında yüksek hızlı bağlantı sağlar.

9. Embedded Systems & ARM Architecture

  • Core Concept: Computers embedded inside devices like washing machines, cars, or smartwatches. Most use ARM-based processors.
    🇹🇷 Temel Mantık: Bilgisayarın masaüstünde değil; çamaşır makinesi, araba veya akıllı saat gibi cihazların içine "gömülmesi" olayıdır.
  • Processor Types:
    Application Processor: General-purpose (e.g., smartphone CPU — does everything).
    Dedicated Processor: Single-task only (e.g., washing machine controller).
    🇹🇷 İşlemci Farkı: Application = Akıllı telefondaki, her işi yapan. Dedicated = Çamaşır makinesindeki, tek göreve adanmış.
  • 3 Rules of Embedded Systems (R.S.C.):
    Real-time — Must respond instantly, no delay tolerance.
    Strict constraints — Low power, small size, cheap.
    Custom designed — Designed specifically for the device.
    🇹🇷 Şifre R.S.C.: Real-time (Gerçek zamanlı, gecikme affedilmez), Strict constraints (Az elektrik, küçük boyut, ucuz), Custom designed (Cihaza özel tasarım).
  • Why ARM? Uses RISC (Reduced Instruction Set Computer) architecture. Simplified instruction set → very low power consumption.
    🇹🇷 ARM Neden Popüler?: RISC mimarisi sayesinde komut seti basitleştirilmiştir, çok az güç tüketir (Low power).

10. Early Computer Structures (IAS & Von Neumann)

  • Von Neumann Architecture: Data and Instructions are stored in the same memory and share the same bus.
    🇹🇷 Von Neumann Kuralı: Veriler ve Komutlar aynı bellekte tutulur ve aynı yolu kullanır.
  • Von Neumann Bottleneck: The CPU cannot fetch an instruction and data simultaneously because there is only one bus.
    🇹🇷 Darboğaz: İşlemci, komutu ve veriyi aynı anda bellekten çekemez çünkü yol tektir.
  • IAS Computer Components (M.A.C.I.):
    Main Memory — Stores instructions and data
    ALU — Arithmetic Logic Unit, performs calculations
    Control Unit — Manages the system
    I/O — Input/Output equipment
    🇹🇷 Şifre M.A.C.I.: Main Memory (Komut ve veri saklar), ALU (Hesaplama yapar), Control Unit (Sistemi yönetir), I/O (Giriş/Çıkış).
  • Critical Registers:
    PC (Program Counter): Holds the address of the next instruction.
    IR (Instruction Register): Holds the current instruction being executed.
    MAR (Memory Address Register): Holds the memory address to be accessed.
    MBR (Memory Buffer Register): Holds the data coming from/going to memory.
    🇹🇷 Hayati Register'lar: PC = Sıradaki komutun adresi, IR = Şu an çalışan komut, MAR = Bellekte gidilecek adres, MBR = Bellekten gelen/giden veri.

11. 🚨 Simple Interrupt Processing (Sınav Favorisi!)

⚠️ Hocanın "Boşluk Doldurma" sormayı en sevdiği tablo algoritmasıdır. İngilizce orijinal metinleri kesinlikle ezberle!

⚙️ HARDWARE (Donanımın Otomatik Yaptıkları)

  • 1. Device controller or other system hardware issues an interrupt
    🇹🇷 Donanım kesme sinyali yollar — "Kargocu zile basar"
  • 2. Processor finishes execution of current instruction
    🇹🇷 İşlemci o an çözdüğü komutu/işlemi bitirir. Anında bırakmaz!
  • 3. Processor signals acknowledgment of interrupt
    🇹🇷 İşlemci "Tamam, duydum geliyorum" diye onay sinyali yollar
  • 4. Processor pushes PSW and PC onto control stack
    🇹🇷 Geri dönebilmek için kalınan adresi (PC) ve durumu (PSW) yığına kaydeder — "Kitaba ayraç koyar"
  • 5. Processor loads new PC value based on interrupt
    🇹🇷 Kesme programının (ISR) adresini PC'ye yükler ve oraya dallanır

💻 SOFTWARE (Yazılımın / İşletim Sisteminin Yaptıkları)

  • 6. Save remainder of process state information
    🇹🇷 Ana programın geriye kalan diğer bilgilerini güvenli bir yere kaydeder
  • 7. Process interrupt
    🇹🇷 Asıl işi yapar! Kargoyu teslim alır, yazıcıya kağıt verir vs.
  • 8. Restore process state information
    🇹🇷 İş bitti! 6. adımda sakladığı o ekstra bilgileri geri yükler
  • 9. Restore old PSW and PC
    🇹🇷 Eski programa geri dönüş biletidir! Masaya geri oturur, ayraç koyduğu sayfayı geri yükler
🧠 Sınav İçin Taktik: Hoca Nereyi Boş Bırakır?

Özellikle mor renkli kutularla işaretlenen 2. , 4. ve 9. adımlara çok dikkat et. Öğrenciler genelde işlemcinin anında komutu yarıda kestiğini düşünür (2. adımda yanılırlar) veya "PC ve PSW"nin (Program Counter ve Program Status Word) yığına kaydedilip (Push) tekrar oradan çağrılmasını (Restore) unuturlar. Bu İngilizce terimleri elinle birkaç kez yazarak pekiştir.
